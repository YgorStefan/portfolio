  <how-to-verify>
With `php artisan serve` running, open http://localhost:8000 and scroll to the contact section.

**Test 1 — Validation errors (CONTACT-02):**
1. Click "Enviar Mensagem" without filling any field
2. Expected: page reloads at #contact, red inline errors appear under each empty field ("The name field is required" etc.)
3. Fill only the name field, leave the rest empty, submit again
4. Expected: name field has no error, other three fields show errors, name value is preserved in the input

**Test 2 — Valid submission + log verification (CONTACT-03, CONTACT-04):**
1. Fill all four fields with valid data: name="Test User", email="test@example.com", subject="Test", message="Hello"
2. Click "Enviar Mensagem"
3. Expected: button shows "Enviando..." briefly, then page redirects to #contact with green "Mensagem enviada com sucesso! Responderei em breve." banner
4. Open `storage/logs/laravel.log` and confirm a log entry exists mentioning the email (look for "ContactFormMail" or the email subject like "[Portfólio] Test")

**Test 3 — PRG pattern — no double submit (CONTACT-02):**
1. After the successful submission in Test 2, press F5 (or Ctrl+R)
2. Expected: browser does NOT show a "resend form data?" dialog — the page simply reloads showing the contact section (no duplicate email sent)

**Test 4 — Submit button disabled state (CONTACT-04):**
1. Fill the form with valid data again and click submit
2. Expected: the submit button briefly shows "Enviando..." text and appears disabled (dimmed/grayed) during the in-flight request

**Test 5 — Rate limiting (CONTACT-05):**
1. Submit the form 6 times in quick succession (valid or invalid data is fine)
2. Expected: the 6th submission returns a "Too Many Requests" (HTTP 429) error page

**Test 6 — Social links (CONTACT-06):**
1. Click the GitHub link → must open https://github.com/ygor-stefankowski in a new tab
2. Click the LinkedIn link → must open https://linkedin.com/in/ygor-stefankowski in a new tab
3. Confirm WhatsApp link is visible (href starts with https://wa.me/)
4. Confirm Email link is visible (href starts with mailto:)
