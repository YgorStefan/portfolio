**Step 1 — Create the app/Mail/ directory and ContactFormMail.php:**

Create `app/Mail/ContactFormMail.php` with this content exactly:

```php
<?php

namespace App\Mail;

use Illuminate\Bus\Queueable;
use Illuminate\Mail\Mailable;
use Illuminate\Mail\Mailables\Address;
use Illuminate\Mail\Mailables\Content;
use Illuminate\Mail\Mailables\Envelope;
use Illuminate\Queue\SerializesModels;

class ContactFormMail extends Mailable
{
    use Queueable, SerializesModels;

    public function __construct(
        public readonly array $formData,
    ) {}

    public function envelope(): Envelope
    {
        return new Envelope(
            replyTo: [
                Address::make($this->formData['email'], $this->formData['name']),
            ],
            subject: '[Portfólio] ' . $this->formData['subject'],
        );
    }

    public function content(): Content
    {
        return new Content(
            view: 'mail.contact',
        );
    }

    public function attachments(): array
    {
        return [];
    }
}
```

Key details:
- `replyTo` on the Envelope means hitting "Reply" in the owner's email client addresses the sender directly — critical UX for a contact form.
- The `from` address is NOT set on the Envelope — it comes from the global `MAIL_FROM_ADDRESS` env var via config/mail.php `from` setting.
- `$this->formData` is a public property — it is automatically available in mail/contact.blade.php as `$formData`.
- `[Portfólio]` prefix on subject helps the owner identify portfolio contacts in their inbox.
- Do NOT use `ShouldQueue` — queues are out of scope for this project (shared hosting, no worker daemon).

```blade
            <td style="padding: 8px 12px; background: #f9f9f9;">{{ $formData['subject'] }}</td>
        </tr>
        <tr>
            <td style="padding: 8px 12px; font-weight: bold; vertical-align: top;">Mensagem:</td>
            <td style="padding: 8px 12px;">{!! nl2br(e($formData['message'])) !!}</td>
        </tr>

    <hr style="margin: 20px 0; border: none; border-top: 1px solid #eeeeee;">

```

Key detail: `{!! nl2br(e($formData['message'])) !!}` — `e()` HTML-encodes the message to prevent XSS, then `nl2br()` converts newlines to `<br>` tags for readable line breaks. The `{!! !!}` (unescaped output) is required because `nl2br()` returns HTML `<br>` tags that must not be double-escaped. Order is critical: escape first, then convert newlines.
