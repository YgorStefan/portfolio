Editar resources/views/layouts/app.blade.php com as seguintes mudanças exatas:

1. Na tag `<html>` (linha 2), adicionar a classe `no-transition` além de `scroll-smooth`:
   ```html
   <html lang="pt-BR" class="scroll-smooth no-transition">
   ```
   Nota: `no-transition` será removida pelo script após o primeiro render. A classe `dark` será adicionada dinamicamente pelo script — não colocar no HTML estático.

2. Antes da linha `@vite(...)`, inserir este bloco de script inline exatamente como escrito:
   ```html
   <script>
       (function() {
           var saved = localStorage.getItem('theme');
           var prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
           var theme = saved || (prefersDark ? 'dark' : 'light') || 'dark';
           if (theme === 'dark') {
               document.documentElement.classList.add('dark');
           }
           // Remover no-transition após o primeiro paint para evitar flash de animação
           requestAnimationFrame(function() {
               requestAnimationFrame(function() {
                   document.documentElement.classList.remove('no-transition');
               });
           });
       })();
   </script>
   ```
   Posicionamento: ANTES do @vite() — o script precisa rodar antes do CSS ser aplicado.

3. Na tag `<body>` (linha 16), substituir as classes hardcoded dark pelo equivalente com fallback light:
   ```html
   <body class="bg-gray-50 dark:bg-gray-950 text-gray-900 dark:text-white antialiased transition-colors duration-150" x-data>
   ```
   Explicação: sem .dark no html = light mode (bg-gray-50, text-gray-900). Com .dark = dark mode (bg-gray-950, text-white).

O arquivo final deve ter na ordem: DOCTYPE, html, head (metas, script inline, @vite, link devicon), body.

2. Após o bloco `@theme { ... }`, adicionar as regras de light mode. O site atual é dark-first — as variáveis em @theme representam o dark mode. O light mode remapeia essas variáveis:

```css
/* Light mode: remapear custom properties quando .dark está ausente no html */
:root:not(.dark) {
    --color-bg-primary: #f9fafb;
    --color-bg-card: #ffffff;
}

/* Dark mode: garantir que as variáveis do @theme sejam aplicadas com .dark */
@variant dark {
    :root {
        --color-bg-primary: #030712;
        --color-bg-card: #111827;
    }
}
```

3. Adicionar transition suave nos elementos que usam as custom properties:

```css
/* Transition suave de 150ms para troca de tema */
html {
    transition: background-color 150ms ease, color 150ms ease;
}

body,
nav,
section,
footer {
    transition: background-color 150ms ease, color 150ms ease, border-color 150ms ease;
}
```

O arquivo final deve manter o `@import "tailwindcss"` no topo, seguido de `@layer base`, depois `.no-transition`, depois `@theme`, depois `:root:not(.dark)`, depois `@variant dark`, depois as transitions, depois as regras do `.swiper-skills`.
