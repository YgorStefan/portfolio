Diagnosticar e corrigir o BUG-01: a aba do Chrome continua girando indefinidamente após o carregamento da página, indicando que algum recurso ou timer está impedindo o evento `load` de ser considerado "completo" pelo browser.

Purpose: Aba girando indefinidamente é o primeiro sinal visual que um recrutador vê ao abrir o portfólio — passa impressão de site quebrado ou lento.

<interfaces>
<!-- Suspeitas documentadas em CONTEXT.md D-01, D-02 -->

Suspeitas a investigar em ordem de probabilidade:
1. Swiper autoplay com loop: true — pode manter o JS engine ativo e sinalizar ao browser que a página não terminou
2. AOS IntersectionObserver — pode manter observers ativos após load
3. Devicon CDN (jsDelivr) — request pendente ou CORS issue bloqueando o evento load
4. Erro JS silencioso — exception não capturada mantendo estado inválido

Config Swiper atual (app.js linhas 21-37):
```js
new Swiper('.swiper-skills', {
    modules: [Navigation, Autoplay],
    loop: true,
    autoplay: {
        delay: 3000,
        disableOnInteraction: false,
    },
    ...
});
```

O `loop: true` + `autoplay` combinados criam um ciclo de timers. Em alguns browsers, isso pode ser interpretado como "página ainda processando". Fix candidato: adicionar `waitForTransition: false` ao autoplay, ou mover a instanciação do Swiper para o evento `load` em vez de `DOMContentLoaded`.

Lógica de diagnóstico: O evento `load` dispara apenas quando TODOS os recursos da página (imagens, scripts, fontes, iframes) terminam de carregar. Se a aba do Chrome não para:
- Network tab: verificar se alguma request está "pending" indefinidamente
- Console: verificar erros de rede ou JS
- Performance tab: verificar timers persistentes (setInterval, RAF loop)
